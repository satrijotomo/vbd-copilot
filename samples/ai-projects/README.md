## CUSTOMER_NAME Intelligent Bill Explainer

> **Note:** `CUSTOMER_NAME` is a placeholder for a real utility company. The project was built for an Italian energy retailer serving 10 million customers. When you run the AI Project agents for your own customer, the output uses the actual company name throughout.

**Customer:** CUSTOMER_NAME (10M customers, Italian energy retail)
**Problem:** Complex energy bills drive high call-center volume and customer churn
**Solution:** A RAG-powered chatbot that explains bills in plain language, with FAQ mode and personalised bill lookup

### Architecture highlights

- **Azure OpenAI** (GPT-4o + GPT-4o-mini) with model routing — simple queries go to mini, complex to full
- **Azure AI Search** (Standard S1, hybrid + semantic ranker) over a knowledge base of tariff docs
- **Azure Cosmos DB** (Serverless) for conversation sessions, messages, and feedback
- **Azure Container Apps** (Consumption, 2–20 replicas) running FastAPI + uvicorn
- **Azure API Management** (Standard v2) for rate limiting and API policies
- **Azure Front Door** (Premium) with WAF and EU geo-filtering
- **Full VNet isolation** with private endpoints for all backend services
