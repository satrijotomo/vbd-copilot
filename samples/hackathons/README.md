## GitHub Copilot for Developers - FinCore Bank (L300, 9 challenges)

**Target audience:** Banking and financial services developer teams
**Format:** Individual or pair work through progressive challenges on a shared codebase
**Environment:** GitHub Codespaces (recommended) or local setup with Python 3.11+, Node.js 22+

Participants inherit a deliberately flawed Python FastAPI banking service and use GitHub Copilot to fix bugs, add tests, refactor across files, build new features, and ship an autonomous PR via the Coding Agent — all in one day.

### Deliberate bugs in the starter code

The starter application ships with four intentional problems that challenges progressively expose and fix:

1. **Float money** — monetary values use Python `float`, introducing rounding errors
2. **No test coverage** — zero tests on day one
3. **Error swallowing** — bare `except:` clauses hide failures silently
4. **No audit logging** — account mutations are not tracked

Work accumulates across the day: code written in Challenge 01 is debugged in Challenge 04 and refactored in Challenge 06.
